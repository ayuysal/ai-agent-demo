Security & Engine
🛡

Hallucination Prevention

Static Data Engine
▶
  • Critical facts are never LLM-generated — injected from verified static sources
  • Immutable reference data layer that cannot be overridden by prompt context
  • Response cross-validation against source-of-truth before delivery
🔒

Data Separation

Isolated Pipelines
▶
  • KB, Web, and Static Data flow through isolated pipelines
  • External data passes through sanitization before reaching LLM context
  • Source attribution on every response — traceable to origin
🔐

Auth & Access Tiers

Public / SSO / RBAC
▶
  • Public tier: KB, FAQs — no auth required
  • SSO tier: Internal docs, customer data — requires authentication
  • Role-based: Admin data only with explicit role verification
📜

Prompt Security

Guardrails & Constraints
▶
  • Hard boundaries on what the agent must never do
  • Immutable security templates stored separately from user prompts
  • Injection-resistant architecture: user input sandboxed from system instructions
🔭

Validation Engine

Pre-delivery Checks
▶
  • Output checked against static reference data before delivery
  • Web-sourced claims flagged with confidence scores
  • Critical data must match verified sources or response is blocked
